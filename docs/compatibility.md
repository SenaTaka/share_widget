# iOS互換性メモ（デプロイメントターゲット: iOS 17.0）

## 前提
- 実装設定は `IPHONEOS_DEPLOYMENT_TARGET = 17.0`（Debug / Release）に統一する。
- 仕様書（`todo.txt`）の対象OS記載は「iOS 17 以降」とし、リリース前に設定値と整合を確認する。

## iOS 17未満で使えないAPIの事前棚卸し

現時点でコードベースに iOS 17 未満で非対応の API 利用はありません（SwiftUI の最小構成のみ）。今後、以下の候補 API を導入する場合は、必ず利用箇所を本書に追記してください。

| 対象領域 | 想定API / 機能 | 最小対応OS | 対応方針 |
| --- | --- | --- | --- |
| SwiftUI | iOS 17以降の新規モディファイア/コンテナ（例: 新しいスクロール制御、アニメーションAPI） | iOS 17 | `@available(iOS 17.0, *)` を明示し、必要に応じて iOS 16 向け代替UIを `#available` で分岐。 |
| Widget | iOS 17以降の Widget 表示最適化 API | iOS 17 | Widget拡張で `if #available(iOS 17.0, *)` を利用し、旧OS向けフォールバック表示を用意。 |
| PencilKit / UIKit連携 | iOS 17で追加された入力・描画補助API | iOS 17 | ViewModel/Facade 層で呼び出しをラップし、利用不可時は既存描画フローへフォールバック。 |

## 実装ルール
1. **導入前チェック**: 新規API採用時に Apple ドキュメントで Availability を確認する。
2. **ガード方針**: デプロイメントターゲットが iOS 17.0 の間は原則ガード不要だが、将来ダウングレードの可能性に備えて、公開インターフェース境界で `@available` / `#available` の適用方針を明文化する。
3. **代替実装方針**: 非対応OSへ展開する場合は、
   - UI: 同等体験を維持できる簡易UIに置き換える
   - データ処理: 同等結果を返す既存実装へ切り替える
   - Widget: 表示項目を縮退してクラッシュを回避する
