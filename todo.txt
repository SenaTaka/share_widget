# 共同手書きメモアプリ 設計書

## 1. 文書情報

* 文書名: 共同手書きメモアプリ 設計書
* 対象プラットフォーム: iOS
* 想定バージョン: iOS 17 以降
* 開発方式: SwiftUI / PencilKit / Core Data / Cloud 同期 / Widget Extension
* 目的: 複数ユーザーで共同編集できる手書きメモアプリを実装し、その最新状態をホーム画面ウィジェットへ表示する

---

## 2. 背景と目的

### 2.1 背景

一般的なメモアプリはテキスト中心であり、手書きの自由度や図示のしやすさに制約がある。一方で、手書きノート系アプリは個人利用に寄るものが多く、複数人で軽く共同編集し、その結果をホーム画面から即座に確認できる体験は限られている。

### 2.2 目的

本アプリは以下を実現する。

* 手書きメモを快適に作成・編集できる
* 複数人で同一メモ群を共有できる
* 共有相手の変更を各端末で同期できる
* 最新状態をウィジェットに表示できる
* ウィジェットから該当ノートを素早く開ける

### 2.3 非目的

初期リリースでは以下は対象外とする。

* ウィジェット内での自由描画
* 超低遅延のライブカーソル共有
* ストローク単位の高度な競合自動マージ
* Web 版 / Android 版
* 音声通話やチャット機能

---

## 3. プロダクト概要

### 3.1 コンセプト

「共同編集できる手書きメモアプリ」+「その最新状態を見せるウィジェット」

### 3.2 提供価値

* 手書きで素早くアイデアを書ける
* 家族・友人・チームで同じメモ空間を共有できる

### 3.3 想定ユースケース

* 家族の買い物メモ共有
* カップルや友人との手書き伝言板
* 学生同士の授業メモ共有
* 小規模チームのホワイトボード代替

## 6. 機能要件

## 6.1 アカウント・利用前提

* ユーザーは iCloud 利用可能であること
* 共有機能は iCloud ベースで動作すること
* 共有相手も同一アプリを導入していることを前提とする

## 6.2 ワークスペース機能

* ワークスペースを新規作成できる
* ワークスペース名を編集できる
* ワークスペース一覧を表示できる
* ワークスペースをアーカイブできる
* ワークスペース単位で共有できる

## 6.3 ノート機能

* ノートを新規作成できる
* ノート一覧を表示できる
* ノートタイトルを編集できる
* ノートを削除できる
* ノートをウィジェット固定対象に設定できる

## 6.4 手書き編集機能

* キャンバス上で描画できる
* Pencil / 指に対応する
* 消しゴム、ペン、マーカー等の基本ツールを使える
* 変更は一定時間無操作後に自動保存される
* 画面離脱時に強制保存される

## 6.5 共同編集機能

* ワークスペース共有リンクを発行できる
* 招待を受諾すると共有ワークスペースへ参加できる
* 他参加者の更新内容を各端末へ同期できる
* 競合発生時にユーザーへ再読込または上書きの選択肢を提示する

## 6.6 ウィジェット機能

* 固定したノートの最新サムネイルを表示する
* タイトル、更新者、更新時刻を表示する
* タップで該当ノート編集画面へ遷移する
* 軽操作としてノート切替を将来追加できる構造にする

## 6.7 同期状態表示

* ローカル保存中
* クラウド同期中
* 最新状態
* 競合あり
* エラー
  をアプリ上で区別表示する

## 7. 非機能要件

### 7.1 性能

* 1回の通常保存は体感上待ちを感じにくいこと
* ノート一覧表示は軽快であること
* ウィジェット表示用画像生成は短時間で完了すること

### 7.2 可用性

* オフラインでもローカル編集できること
* 通信復帰後に同期できること

### 7.3 拡張性

* 将来ストローク単位同期へ移行可能な構造とする
* 将来コメント機能や複数ウィジェットへ拡張可能とする

### 7.4 保守性

* UI / Domain / Data / WidgetBridge を分離する
* 永続化実装と表示実装を疎結合にする

### 7.5 セキュリティ

* 共有は招待ベースを基本とする
* 表示用キャッシュと編集用真実データを分離する
* ウィジェット側は表示に必要な最小データのみ参照する

---

## 8. 全体アーキテクチャ

### 8.1 構成方針

アプリは以下の層に分ける。

* Presentation 層
* Domain 層
* Data 層
* Widget Bridge 層
* Widget Extension

### 8.2 役割分担

#### Presentation 層

* 画面表示
* ユーザー操作受付
* ViewModel による状態制御

#### Domain 層

* ユースケース定義
* ドメインモデル定義
* Repository インターフェース定義

#### Data 層

* Core Data 永続化
* Cloud 同期
* 共有処理
* PencilKit データ変換

#### Widget Bridge 層

* ウィジェット用 JSON / 画像キャッシュ生成
* App Group への書き出し
* Widget 更新要求

#### Widget Extension

* App Group 内の表示データ読込
* Widget UI 描画
* Deep Link 起動

### 8.3 全体データフロー

1. ユーザーがアプリ内で手書きする
2. 描画データをローカル保存する
3. クラウド同期基盤が変更を他端末へ配布する
4. 保存完了後に表示用プレビュー画像を生成する
5. App Group にウィジェット用データを書き出す
6. Widget を再読込してホーム画面へ最新状態を反映する

---

## 9. データモデル設計

### 9.1 モデル方針

* 共有単位は Workspace とする
* Workspace 配下に複数 Note を持つ
* 初期リリースでは 1 Note = 1 手書きデータ blob とする
* 表示用画像は派生データとして別管理する

### 9.2 エンティティ一覧

#### Workspace

* id: UUID
* name: String
* createdAt: Date
* updatedAt: Date
* ownerUserID: String
* isArchived: Bool
* coverNoteID: UUID?

#### Note

* id: UUID
* workspaceID: UUID
* title: String
* drawingData: Data
* canvasWidth: Double
* canvasHeight: Double
* revision: Int64
* previewVersion: Int64
* updatedAt: Date
* updatedByUserID: String
* isDeleted: Bool
* pinnedForWidget: Bool

#### MemberCache（任意）

* id: UUID
* workspaceID: UUID
* userID: String
* displayName: String?
* permission: String
* lastSeenAt: Date?

### 9.3 ドメインルール

* Note は必ず 1 つの Workspace に属する
* Widget に固定できるノートは 1 件を基本とする
* revision は保存ごとに増加する
* updatedByUserID は最終更新者を示す

---

## 10. 永続化・同期設計

### 10.1 方針

* local-first
* ローカル保存を正とし、クラウドへ非同期反映する
* 他ユーザー共有はクラウド共有機構で実現する

### 10.2 ストア分離方針

* Private Store: 自分が所有するデータ
* Shared Store: 他人から共有されたデータ

### 10.3 同期トリガー

* 描画停止後の debounce 保存
* 画面離脱時保存
* アプリ background 遷移時保存

### 10.4 同期イベント処理

* 保存開始
* 保存完了
* クラウド反映開始
* リモート変更取込完了
* エラー検知

### 10.5 オフライン時の扱い

* ローカル編集は継続可
* 同期待ち状態を UI 表示
* 通信復帰後に順次同期

---

## 11. 共同編集・競合設計

### 11.1 初期リリース方針

* 高度な同時編集統合は行わない
* last-write-wins を基本とする
* ただし無自覚上書きを避ける soft guard を入れる

### 11.2 soft guard の仕様

* 編集開始時の revision を保持する
* 保存時に最新 revision と比較する
* 差異ありなら競合とみなす
* ユーザーへ以下を提示する

  * 最新を読み込む
  * 自分の内容で上書きする

### 11.3 将来拡張

* Stroke 単位の差分保存
* 削除 tombstone 管理
* 競合自動マージ

## 14. ユースケース設計

### 14.1 ワークスペース作成

1. ユーザーが新規作成を押す
2. 名前を入力する
3. Workspace を生成する
4. 一覧へ反映する

### 14.2 ノート作成

1. ワークスペース内で新規ノート作成
2. 空の描画データを持つ Note を作成
3. 編集画面へ遷移

### 14.3 手書き保存

1. ユーザーが描く
2. 入力停止後に autosave 発火
3. 描画データ保存
4. revision 更新
5. プレビュー生成
6. Widget 用キャッシュ更新

### 14.4 共有開始

1. ユーザーが共有を押す
2. 共有対象 Workspace を指定
3. 共有リンク発行
4. 招待送信

### 14.5 共有受諾

1. 相手が共有リンクを開く
2. 共有参加処理を実行
3. ワークスペースを取込む
4. 一覧へ反映

### 14.6 リモート更新反映

1. 他ユーザーが編集・保存
2. 同期基盤が変更を取込む
3. ローカル DB 更新
4. UI と Widget を更新

## 16. エラー設計

### 16.1 想定エラー

* ローカル保存失敗
* 同期失敗
* 共有招待受諾失敗
* 表示用画像生成失敗
* App Group 書込失敗
* 競合発生

### 16.2 エラー方針

* UI 上に人間が理解できる文言で表示する
* リトライ可能なものは再試行導線を出す
* 保存失敗時は破棄せずメモリ保持する
* 同期失敗時もローカル編集結果は残す

## 21. 最終方針

本アプリは、共同編集可能な手書きメモを中核とし、ウィジェットはその結果を即座に確認するための表示窓として扱う。編集の本体と表示キャッシュを分離し、まずは実装負荷と安定性のバランスが取れた MVP を成立させる。そのうえで、競合解決や共同編集の高度化は次段階で拡張する。

この方針により、初期段階で破綻しない現実的な設計と、将来的な拡張余地の両立を図る。
